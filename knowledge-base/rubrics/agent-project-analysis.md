# Rubric: Open-Source Agent Project Analysis

Use this rubric before drafting project profiles or long-form essays.

## Required Dimensions

1. **Goal and user model**: what kind of agent/application does the project
   optimize for?
2. **Control flow**: loop, graph, workflow, conversation, planner/executor,
   event stream, or hybrid.
3. **State and memory**: what persists, what is ephemeral, what is replayable?
4. **Tool model**: tool schema, permissions, sandbox, error handling, retries.
5. **Human control**: approval points, interrupts, supervision, auditability.
6. **Evaluation**: tests, traces, benchmarks, offline eval, production feedback.
7. **Extension model**: plugins, custom nodes, custom agents, integrations.
8. **Operational model**: deployment, concurrency, persistence, cost control.
9. **Failure modes**: prompt drift, tool misuse, infinite loops, brittle memory,
   poor observability, security boundary leaks.
10. **Transferable lessons**: what should other agent builders copy or avoid?

## Evidence Standard

- Static project claims need repository/docs sources.
- Dynamic claims need current verification or explicit timestamp.
- Cross-project claims need at least three project examples.
- Interpretations must be labeled as interpretation, not fact.
