## 图 1：控制流显式光谱五极定位

- **用途**：§3 控制流显式光谱的核心插图，展示五个项目在"黑盒→白盒"光谱上的相对位置

  - 右中：MAF（分层节点，三层嵌套，标注 "Tier 0/1/2 + Workflow"）

- **风格**：扁平化技术插图，深色背景（#1a1a2e）+ 高对比度节点色（蓝/青/紫/橙/绿）
- **比例**：16:9

## 图 4：继承/断裂矩阵可视化

- **用途**：§6 继承与断裂矩阵的核心插图，是本文最稀缺的视角呈现
- **构图**：左右对称布局。左侧为"继承"（绿色调），右侧为"断裂/重构"（橙/红色调）。中间有一条粗竖线分隔，线上方标注 "MAF vs AutoGen"。AutoGen 在左下，MAF 在右上，中间有双向箭头
- **关键元素**：
  - 继承侧（5 项）：
    - 编排模式（Magentic/GroupChat）
    - 双语言支持（Python + .NET）
    - Actor 模型运行时
    - MCP 协议支持
    - 核心概念命名
  - 断裂侧（8 项）：
    - 包结构（三层 → 扁平 Tier）
    - 工作流（无 → Graph + checkpoint）
    - 持久化（无 → DurableTask）
    - 协议（MCP → MCP+A2A+AG-UI）
    - UI（Studio → DevUI）
    - 代码执行（Docker → CodeAct/Hyperlight）
    - 中间件（无 → 原生 Middleware）
    - License（CC-BY-4.0 → MIT）
  - 底部横幅："研究院架构 → 产品团队工程"（用两个不同风格的建筑图标表示）
- **风格**：信息图（infographic）风格，清晰的分栏和编号，深色背景 + 高对比度文字
- **比例**：16:9
- **避免项**：过于拥挤（需保持呼吸感）、3D 效果、无编号的无序列表

## 图 5：五维雷达图

- **用途**：§7 跨维度全景对比的可视化，让读者一眼看到五个项目的相对强弱
- **构图**：五维雷达图，五个轴分别为：控制流显式度 / 角色语义深度 / 生态锁定强度 / 跨语言支持 / 生产就绪梯度。五个项目用不同颜色线条绘制
- **关键元素**：
  - 轴 1（控制流显式度）：LangGraph 最外，smolagents 最内
  - 轴 2（角色语义深度）：CrewAI 最外，smolagents/LangGraph 最内
  - 轴 3（生态锁定强度）：MAF 最外，smolagents 最内
  - 轴 4（跨语言支持）：MAF 最外（真正双语言），其余 Python-only
  - 轴 5（生产就绪梯度）：LangGraph/CrewAI 较外（checkpoint/observability 成熟），MAF 中段（核心稳但扩展新），smolagents 较内（无持久化/可观测性），AutoGen 最内（维护模式）
  - 图例清晰标注颜色对应的项目
- **风格**：数据可视化风格，白色/浅色背景，线条粗细 2-3pt，半透明填充
- **比例**：1:1（方形，适合居中展示）
- **避免项**：3D 雷达图、过多网格线、颜色相近导致难以区分

## 图 6："极简框架"的设计张力循环

- **用途**：§8 跨项目共同困境的概念图，展示五个项目都面临的结构性张力
- **构图**：循环图（circular diagram），五个项目围绕中心主题"极简承诺 vs 功能膨胀"排列
- **关键元素**：
  - 中心：一个被拉扯的弹簧图标，一端标"极简承诺"，一端标"功能膨胀"
  - 外围五个节点，每个节点包含项目名 + 其"极简口号" + "现实反差"：
    - smolagents："~1,000 行" → 1,814 行
    - LangGraph："轻量状态机" → 518 MB 仓库
    - CrewAI："lean, lightning-fast" → 519 文件 / 15+ 子系统
    - AutoGen："简洁多 Agent" → 三层架构 + 跨语言运行时
    - MAF："简洁 Tier API" → 20+ 子包 + 三协议支持
  - 循环箭头：每个节点之间用箭头连接，表示"一个框架的膨胀会推高其他框架的功能预期"
- **风格**：概念图风格，略带讽刺意味的插画风格（但保持技术严肃性），暖灰背景
- **比例**：16:9
- **避免项**：过于卡通化、具体人物肖像、负面情绪表达

## 封面图：五极争霸概念封面

- **用途**：文章封面/社交媒体头图
- **构图**：暗色背景上的抽象"五极"概念。五个发光的几何体（代表五个框架）围绕一个中心坐标系排列，坐标轴隐约可见。几何体之间有能量线/数据流连接，表示竞争与影响关系
- **关键元素**：
  - 中心：抽象坐标系（类似雷达图的 5 个轴），发出微弱蓝光
  - 五极（按大致 star 数大小排列，但不精确对应）：
    - AutoGen：最大几何体，但颜色偏灰（维护模式）
    - CrewAI：次大，暖橙色发光
    - LangGraph：中等，冷蓝色发光
    - smolagents：较小但亮度高（新星），青绿色发光
    - MAF：最小但增长箭头向上，紫色发光
  - 背景：深空/数字空间感，有微弱的代码/数据流纹理
  - 标题文字区域留白（左下或上方，供后期叠加文字）
- **风格**：科幻感技术插画，暗色调（#0d1117 类 GitHub dark 背景），霓虹色高亮，电影海报级构图
- **比例**：16:9（或 2:1 适合 Twitter/公众号封面）
- **避免项**：具体框架 logo（版权风险）、人物、过于复杂的细节（封面需要远距离可读）
