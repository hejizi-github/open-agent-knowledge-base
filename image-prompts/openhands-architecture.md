## 图 3：EventStream 消息拓扑流程图

**用途**: §3.2 末尾，展示 Agent → AgentController → EventStream → Runtime → Frontend 的事件流。

**画面构图**: 中心 hub-spoke 拓扑，中间为 EventStream 总线（圆形 hub）。
- 左侧：Agent（人形图标）→ AgentController（齿轮图标）→ 蓝色箭头（标注"Action"）→ EventStream
- 中间：EventStream 为大型圆形 hub，内部有脉冲波纹效果，标注"EventStream pub/sub"
- 右侧：EventStream → 绿色箭头（标注"Observation"）→ Runtime（Docker 容器图标）→ 绿色箭头 → EventStream → Frontend（浏览器窗口图标）
- 底部：Frontend → 蓝色箭头（标注"用户 Action"）→ EventStream
- 中心圆形上方：用红色小字标注"无中央调度器"

**关键元素**: Agent、AgentController、EventStream hub、Runtime、Frontend、蓝色 Action 流、绿色 Observation 流、"无中央调度器"标注。

**风格**: 技术架构图风格，类似 cloud architecture diagram。蓝色系（Action）和绿色系（Observation）区分事件类型。背景浅灰。

**比例**: 16:9（横版）

**避免项**: 避免使用 UML 标准符号（太学术）、避免箭头交叉混乱、避免 hub 过小。

## 图 5：Runtime-Sandbox 安全边界示意图

**用途**: §4.3 末尾，对比 Docker Sandbox 模式和 local 模式的执行边界。

**画面构图**: 左右对比，中间有切换控制杆。
- 左侧（Docker Sandbox 模式）：
  - 顶部标注"Docker Sandbox 模式"（蓝色标签）
  - Agent → Runtime → Docker 容器（立方体图标，有锁标志）
  - 容器内部：文件系统树、网络隔离图标、标注"隔离的文件系统 + 独立网络命名空间"
  - 底部标注"适用场景：生产隔离"
- 右侧（local 模式）：
  - 顶部标注"local 模式"（橙色标签）
  - Agent → Runtime → 直接指向宿主机（服务器图标）
  - 宿主机上：文件系统、网络端口、标注"直接操作宿主机文件系统和网络"
  - 底部标注"适用场景：开发迭代"
- 中间：双向箭头，标注"RUNTIME=docker / local"，箭头上方有一个切换开关图标

**关键元素**: Docker 容器（带锁）、宿主机服务器、切换开关、两种模式下的文件系统/网络表示、适用场景标签。

**风格**: 安全架构图风格，类似 cloud security diagram。Docker 侧用冷色调（蓝/灰），local 侧用暖色调（橙/黄）。

**比例**: 16:9（横版）

**避免项**: 避免过于抽象的几何图形、避免安全边界线条不清晰、避免两种模式的视觉权重不对称。

## 图 7：封面图

**用途**: 文章头部封面，用于社交媒体分享和文章列表缩略图。

**画面构图**: 宽幅场景，中心偏右放置。
- 背景：抽象的代码宇宙——深色背景（深蓝/近黑），散布着发光的代码片段（Python、YAML、Dockerfile 的模糊代码行），像星云一样漂浮。
- 中心主体：一只半机械半有机的"手"（呼应 OpenHands 名称），手掌向上托举着一个发光的 Docker 容器（立方体，内部有代码流动的效果）。手的机械部分有精密的齿轮和电路纹理，有机部分有皮肤质感。
- 手的周围：EventStream 的事件流用发光粒子表示，蓝色粒子（Action）从左向右流动，绿色粒子（Observation）从右向左流动，形成环绕手的轨道。
- 底部：淡入的文章标题区域（留给后期添加文字），有轻微的渐变遮罩。
- 氛围：科技与人文的交汇，既硬核（代码、Docker、机械）又温暖（手、有机纹理、柔和光效）。

**关键元素**: 半机械半有机的手、Docker 容器（发光立方体）、代码星云背景、EventStream 粒子流（蓝/绿）、齿轮/电路纹理。

**风格**: 数字艺术/概念艺术，类似 WIRED 杂志封面或技术大会主视觉。高质量渲染，有景深效果，电影级光影。

**比例**: 21:9（超宽横版，适合横幅/头图）

**避免项**: 避免出现具体的人脸（手的其他部分可以抽象化）、避免文字（纯图，文字后期叠加）、避免过于卡通/低多边形风格、避免出现 OpenHands 项目 logo（避免版权问题）。
