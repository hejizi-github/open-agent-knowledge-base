# Task Framing

## Goal
调研 1-2 个真实开源 Agent 项目，获取当前仓库状态、核心架构、维护活跃度等 ground truth，产出 `.evolve/library/facts/<id>.md` 事实条目，为后续深度长文写作积累可验证素材。

## Wiki Pages Used
- `.evolve/library/methodology/_form.md` — 意图锚定与形态识别；确认本轮产物为 facts 库增量
- `.evolve/library/methodology/deep-tech-essay-agent-architecture-001.md` — 深度长文方法论；反共识点需要真实项目数据支撑
- `.evolve/library/methodology/reverse-anthropic-building-effective-agents.md` — Anthropic 模式归纳法；本轮选取的项目应能映射到其 5+1 模式
- `.evolve/library/methodology/reverse-langchain-what-is-an-agent.md` — 光谱定义法；本轮项目应体现不同 agentic 程度

## Expert Lens
- Anthropic 视角：Agent 系统的复杂度分层（building block → workflow → agent）；简单 > 复杂是实证结论而非口号
- LangChain 视角：Agent 不是二元概念，而是光谱；框架的价值在于显式控制流而非黑盒编排
- 本轮新增：需用真实仓库数据验证上述框架在实际开源项目中的映射关系

## Default Assumptions
- 选取 Hugging Face `smolagents`（极简工具调用型）和 `langgraph`（显式状态机编排型）作为首轮调研对象
- 二者设计哲学差异大，可形成对比；若 API 数据获取失败则 fallback 到 CrewAI
- 所有项目状态数据以 GitHub API 返回时间戳为准，不凭记忆写 star/release 数

## This Session Step
Step C：事实/环境调研。上一轮已完成 Step B（方法论学习），methodology 库已覆盖深度长文写作框架，但 facts 库为空。在缺乏真实项目 ground truth 的情况下进入 Step D 会导致文章断言无据，因此本轮只补 facts。
