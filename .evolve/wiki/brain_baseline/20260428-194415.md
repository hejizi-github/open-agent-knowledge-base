# 脑内默认假设：Agent 记忆系统（形态识别前）

## 1. 关于"记忆"概念的默认假设

1. **Agent 的记忆 = LLM 的上下文窗口**：默认认为只要给 LLM 足够长的上下文，Agent 就能"记住"之前的事情。更长的上下文窗口（128K→1M→10M tokens）会让记忆问题自然消失。

2. **RAG 就是 Agent 的长期记忆**：默认认为把对话历史存入向量数据库，然后在每次调用时检索相关片段，这就是 Agent 的"长期记忆"。RAG 和记忆的区别只是技术实现细节。

3. **记忆是框架的"附加功能"**：默认认为记忆层是框架的可选组件——框架"支持"记忆（通过向量数据库集成）就够了，记忆不是核心架构问题。

4. **"有状态" = "有记忆"**：默认认为任何能在多次调用之间保留 state 的系统就是有记忆的。checkpoint、session storage、conversation history 都是"记忆"的不同名称。

## 2. 关于各框架记忆实现的默认假设

5. **LangGraph 的 checkpoint 是"最好的记忆"**：因为 LangGraph 原生支持状态持久化，所以它的记忆实现是行业标杆。其他框架（如 smolagents）"不支持记忆"是因为设计简单。

6. **Mem0 是一个"记忆增强库"**：默认认为 Mem0 只是在向量数据库上封装了一层 API，把对话历史自动编码和检索。没有本质创新。

7. **CrewAI 的 memory + knowledge + RAG 三个子系统是"全面"的**：默认认为 CrewAI 把记忆分成多个子系统是一种周到的设计，覆盖的场景更广。

## 3. 关于记忆评估的默认假设

8. **记忆的评估标准是"检索准确率"**：默认认为只要检索系统能召回相关的历史信息，记忆就是"好"的。没有考虑过"该记什么、不该记什么"这个筛选问题。

9. **记忆系统的性能瓶颈在向量数据库**：默认认为记忆延迟主要来自向量检索（ANN search），优化方向是更快的向量数据库或更小的 embedding 模型。

## 4. 关于文章结构的默认假设

10. **文章应该按"记忆类型"组织**：短期记忆→中期记忆→长期记忆，或者按技术方案（向量数据库、图数据库、键值存储）组织。

---

**以上假设未经外部来源验证，仅用于 Step A 形态识别时的 diff 基线。**

**预计会与被推翻的假设**：
- #1（上下文≠记忆）和 #2（RAG≠记忆）是最可能被推翻的，因为 Mem0 和 Letta 的架构明确区分了这些概念
- #3（记忆是附加功能）可能被推翻，因为 Letta 的 memory_blocks 是核心抽象而非附加组件
- #8（检索准确率）可能被推翻，因为 Mem0 的 benchmark 包含了"该记什么"的评估维度（LoCoMo, LongMemEval）
