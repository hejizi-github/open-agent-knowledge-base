# Microsoft Agent Framework (MAF) — 项目事实卡

## 1. 项目身份

- **全称**：Microsoft Agent Framework (MAF)
- **定位**：A comprehensive multi-language framework for building, orchestrating, and deploying AI agents（构建、编排和部署 AI Agent 的多语言综合框架）
- **发起方**：Microsoft（微软官方团队，非研究院独立项目）
- **创建时间**：2025-04-28（截至 2026-04-28 刚满一年）
- **当前状态**：**活跃开发中**，微软官方主推的 Agent 框架
- **License**：MIT（比 AutoGen 的 CC-BY-4.0 更商业化友好）
- **GitHub**：https://github.com/microsoft/agent-framework
- **文档站**：https://learn.microsoft.com/agent-framework/
- **主页**：https://aka.ms/agent-framework

## 4. 架构设计

### 4.1 三层导入体系（Tier 0/1/2）

MAF 采用**扁平 namespace 包 + 分层导入**设计，与 AutoGen 的 strict Core/Chat/Ext 三层不同：

```
┌─────────────────────────────────────────────┐
│  Tier 2 — Vendor Connectors                 │  ← 按厂商分组的可选子包
│  from agent_framework.openai import ...     │     openai, azure, google, anthropic
│  from agent_framework.foundry import ...    │     foundry, gemini, ollama, bedrock
├─────────────────────────────────────────────┤
│  Tier 1 — Advanced Components               │  ← 高级功能组件
│  from agent_framework.vector_data import ...│     vector_data, observability, exceptions
├─────────────────────────────────────────────┤
│  Tier 0 — Core Agent API                    │  ← 基础 Agent 和工具
│  from agent_framework import Agent, ai_function│   Agent, tools, types, middleware, telemetry
└─────────────────────────────────────────────┘
```

**设计目标**：Developer experience is key（`docs/design/python-package-setup.md` 首句）。Tier 0 组件无需深层导入即可使用。

### 4.2 Namespace Package 策略

- 子包按 **vendor** 分组（而非按类型），如 `agent_framework.google` 包含 Google 所有连接器
- 支持 **lazy loading**，避免导入整个包
- 各子包有**独立生命周期和版本**，通过 namespace 包实现
- 非 GA 依赖/实验性功能隔离在独立子包中，避免污染主包

### 4.3 编排模式（Orchestrations）

`agent-framework-orchestrations` 包提供高层工作流构建器：

| Builder | 模式描述 | 与 AutoGen 的对应关系 |
|---------|---------|---------------------|
| `SequentialBuilder` | 顺序链式执行 | 无直接对应（新增） |
| `ConcurrentBuilder` | 并发 fan-out，聚合结果 | 类似 GraphFlow 并发 |
| `HandoffBuilder` | 去中心化路由，Agent 自主决定交接目标 | 新增（类似 OpenAI Swarm） |
| `GroupChatBuilder` | 选择器驱动的群聊 | 继承 `SelectorGroupChat` |
| `MagenticBuilder` | Magentic One 结构化编排 | 继承 `MagenticOneGroupChat` |

**关键继承证据**：MagenticBuilder 和 GroupChatBuilder 的命名与 AutoGen v0.7 的 Teams 类型完全一致，且文档明确引用相同概念。

### 4.4 工作流（Workflows）

MAF 引入了 **Graph-based Workflows**，这是 AutoGen 所不具备的 LangGraph 式能力：

- **Data flows**：通过数据流连接 Agent 和确定性函数
- **Streaming**：工作流事件流式输出
- **Checkpointing**：状态检查点，支持中断恢复
- **Human-in-the-loop**：人工审批介入点
- **Time-travel**：回溯到任意检查点重新执行

### 4.5 Durable Agents / Durable Workflows

通过 `agent-framework-durabletask` 集成微软 Durable Task 框架：

- 持久化 Agent 状态
- 自动恢复对话历史
- 故障自动恢复
- 支持分布式执行

## 5. 核心概念

### 5.1 Agent 类型

```python
from agent_framework import Agent
from agent_framework.openai import OpenAIChatClient

agent = Agent(
    client=OpenAIChatClient(),
    name="MyAgent",
    instructions="You are a helpful assistant.",
    tools=[get_weather, get_menu_specials]
)
```

- `Agent` 是统一入口（Tier 0），所有 Agent 变体都基于此
- 支持 `ai_function` 装饰器简化工具定义
- 可通过 `workflow.as_agent()` 将工作流包装为 Agent

### 5.2 Chat Client 体系

MAF 将 LLM 调用抽象为 **ChatClient**，支持直接独立使用：

```python
from agent_framework.openai import OpenAIChatClient
from agent_framework import Message, Role

client = OpenAIChatClient()
response = await client.get_response([Message("system", [...]), Message("user", [...])])
```

**支持的提供商**：OpenAI, Azure OpenAI, Microsoft Foundry, Anthropic, Gemini, Ollama, Bedrock, GitHub Copilot, Azure AI Agents 等。

### 5.3 Middleware 系统

灵活的请求/响应中间件管道：
- 请求/响应处理
- 异常处理
- 自定义管道逻辑
- 在 Python 和 .NET 中均有实现

### 5.4 协议支持矩阵

| 协议 | 支持状态 | 说明 |
|------|---------|------|
| **MCP** | 已支持 | Model Context Protocol，工具/上下文标准化 |
| **A2A** | 已支持（v1.2.0+） | Google 的 Agent-to-Agent 开放标准，`agent-framework-a2a` 包 |
| **AG-UI** | 已支持 | Agent-User Interaction 协议，与 LangGraph/CrewAI/Pydantic AI 互操作 |

## 6. 工具与扩展生态

### 6.1 DevUI

交互式开发和调试 UI（`agent-framework-devui`）：
- 可视化 Agent 开发、测试和调试
- 工作流可视化
- 与代码开发流程集成

### 6.2 AF Labs

实验性功能包（`agent-framework-lab`）：
- 基准测试工具
- 强化学习实验
- 研究倡议

### 6.3 代码执行

- **CodeAct**（ADR 0024，proposed）：通过 Hyperlight 沙箱执行模型生成的代码
- 模型写可执行代码而非固定 JSON schema，在受控运行时中调用工具
- 支持多步逻辑、条件分支、数据转换，减少延迟和 token 成本

### 6.4 内存与搜索

- Redis (`agent-framework-redis`)
- Mem0 (`agent-framework-mem0`)
- Azure AI Search (`agent-framework-azure-ai-search`)
- Azure Cosmos DB (`agent-framework-azure-cosmos`)

## 9. 与 AutoGen 的继承与断裂

### 继承关系

| 方面 | MAF 继承内容 |
|------|------------|
| 编排模式 | MagenticBuilder、GroupChatBuilder 直接继承 AutoGen Teams 概念 |
| 双语言支持 | Python + .NET 延续 AutoGen Core 的跨语言设计 |
| Actor 模型 | 运行时设计延续 AutoGen Core 的异步消息传递思想 |
| MCP 支持 | 延续 AutoGen 的 McpWorkbench 思路 |
| 概念命名 | Agent, Team/GroupChat, Tool, Memory 等核心概念保持一致 |

### 断裂/重构

| 方面 | MAF 的变化 |
|------|-----------|
| 包结构 | AutoGen 的 strict 三层（Core/Chat/Ext）→ MAF 的扁平 Tier 0/1/2 + namespace packages |
| 工作流 | AutoGen 无原生图工作流 → MAF 新增 Graph-based Workflows（checkpointing, time-travel） |
| 持久化 | AutoGen 无持久化 → MAF 原生 DurableTask 集成 |
| 协议 | AutoGen 仅 MCP → MAF MCP + A2A + AG-UI 三协议 |
| UI | AutoGen Studio（非生产就绪）→ MAF DevUI（集成开发调试） |
| 代码执行 | AutoGen DockerCommandLineCodeExecutor → MAF CodeAct + Hyperlight 沙箱 |
| 中间件 | AutoGen 无中间件概念 → MAF 原生 Middleware 系统 |
| License | CC-BY-4.0 → MIT（更商业化友好） |

## 10. 已知限制与注意事项

1. **项目极新**：2025-04 创建，API 可能仍有 breaking changes（commits 中可见 `[Breaking]` 标记）
2. **CodeAct 尚未落地**：ADR 0024 状态为 proposed，非 accepted
3. **orchestrations 包为 pre-release**：`pip install agent-framework-orchestrations --pre`
4. **durabletask 包为 pre-release**：同上
5. **默认锁定 Microsoft Foundry**：quickstart 示例默认使用 Azure CLI + Foundry，非 OpenAI
6. **第三方系统免责声明**：README 明确声明使用非 Azure 模型/服务需自行承担风险
