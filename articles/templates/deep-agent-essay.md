## Reader Promise

读完后，读者应该能：

- 理解几个开源 Agent 项目的架构差异；
- 判断哪些设计适合自己的产品/工程场景；
- 避免至少 3 个常见 Agent 系统设计误区。

## Outline

1. 问题背景：为什么这个问题重要
2. 开源项目地图：项目定位和演化脉络
3. 架构主线：控制流、状态、工具、记忆、评估
4. 项目深拆：每个项目的核心设计和边界
5. 横向对比：相同问题的不同答案
6. 失败模式：哪些设计容易在真实场景里崩
7. 实践建议：如果今天要做 Agent runtime 应该怎么选型
8. 图片提示词：封面、架构图、对比图、流程图
9. Sources
